= 보안 이벤트에 대한 알림 추가
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Unified Manager에서 수신한 다른 이벤트와 마찬가지로 개별 보안 이벤트에 대한 알림을 구성할 수 있습니다.  또한, 모든 보안 이벤트를 동일하게 처리하고 같은 사람에게 이메일을 보내려는 경우 보안 이벤트가 발생할 때마다 알려주는 단일 알림을 만들 수 있습니다.

.시작하기 전에
애플리케이션 관리자 또는 스토리지 관리자 역할이 있어야 합니다.

아래 예에서는 "Telnet Protocol Enabled" 보안 이벤트에 대한 알림을 만드는 방법을 보여줍니다.  클러스터에 대한 원격 관리 액세스를 위해 Telnet 액세스가 구성된 경우 경고가 전송됩니다.  동일한 방법을 사용하여 모든 보안 이벤트에 대한 알림을 생성할 수 있습니다.

.단계
. 왼쪽 탐색 창에서 *저장소 관리* > *알림 설정*을 클릭합니다.
. *알림 설정* 페이지에서 *추가*를 클릭합니다.
. *알림 추가* 대화 상자에서 *이름*을 클릭하고 알림의 이름과 설명을 입력합니다.
. *리소스*를 클릭하고 이 알림을 활성화하려는 클러스터 또는 클러스터를 선택합니다.
. *이벤트*를 클릭하고 다음 작업을 수행하세요.
+
.. 이벤트 심각도 목록에서 *경고*를 선택합니다.
.. 일치하는 이벤트 목록에서 *Telnet 프로토콜 사용*을 선택합니다.


. *작업*을 클릭한 다음 *다음 사용자에게 알림* 필드에서 알림 이메일을 받을 사용자의 이름을 선택합니다.
. 이 페이지에서 알림 빈도, SNMP 탭 발행, 스크립트 실행에 대한 다른 옵션을 구성합니다.
. *저장*을 클릭하세요.
