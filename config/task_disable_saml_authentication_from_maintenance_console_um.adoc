= 유지 관리 콘솔에서 SAML 인증 비활성화
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Unified Manager GUI에 액세스할 수 없는 경우 유지 관리 콘솔에서 SAML 인증을 비활성화해야 할 수도 있습니다.  이는 구성이 잘못되었거나 IdP에 액세스할 수 없는 경우 발생할 수 있습니다.

.시작하기 전에
유지 관리 사용자로서 유지 관리 콘솔에 액세스할 수 있어야 합니다.

SAML 인증이 비활성화되면 Active Directory나 LDAP와 같은 구성된 디렉터리 서비스 공급자가 로그온 인증을 수행합니다.  구성된 원격 사용자 외에도 로컬 사용자와 유지 관리 사용자는 그래픽 사용자 인터페이스에 액세스할 수 있습니다.

UI의 설정/인증 페이지에서 SAML 인증을 비활성화할 수도 있습니다.

[NOTE]
====
SAML 인증이 비활성화되면 Unified Manager가 자동으로 다시 시작됩니다.

====
.단계
. 유지 관리 콘솔에 로그인합니다.
. *메인 메뉴*에서 *SAML 인증 비활성화* 옵션에 대한 번호를 입력합니다.
+
SAML 인증을 비활성화하고 Unified Manager를 다시 시작할 것인지 확인하는 메시지가 표시됩니다.

. *y*를 입력한 다음 Enter 키를 누르면 Unified Manager가 다시 시작됩니다.


다음에 원격 사용자가 Unified Manager 그래픽 인터페이스에 액세스하려고 하면 IdP 로그인 페이지 대신 Unified Manager 로그인 페이지에 자격 증명을 입력하게 됩니다.

필요한 경우 IdP에 액세스하여 Unified Manager 서버 URL과 메타데이터를 삭제합니다.
