= 이벤트 확인 및 해결
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
이벤트를 생성한 문제에 대한 작업을 시작하기 전에 이벤트를 확인해야 반복적으로 알림 알림을 받지 않을 수 있습니다.  특정 이벤트에 대한 시정 조치를 취한 후에는 해당 이벤트를 해결됨으로 표시해야 합니다.

.시작하기 전에
운영자, 애플리케이션 관리자 또는 스토리지 관리자 역할이 있어야 합니다.

여러 이벤트를 동시에 확인하고 해결할 수 있습니다.

[NOTE]
====
정보 이벤트를 확인할 수 없습니다.

====
.단계
. 왼쪽 탐색 창에서 *이벤트 관리*를 클릭합니다.
. 이벤트 목록에서 다음 작업을 수행하여 이벤트를 확인합니다.
+
|===
| 만약 당신이 원한다면... | 이렇게 하세요... 


 a| 
단일 이벤트를 확인하여 해결됨으로 표시합니다.
 a| 
.. 이벤트 이름을 클릭하세요.
.. 이벤트 세부 정보 페이지에서 이벤트의 원인을 확인하세요.
.. *확인*을 클릭하세요.
.. 적절한 시정 조치를 취하세요.
.. *해결됨으로 표시*를 클릭하세요.




 a| 
여러 이벤트를 확인하여 해결됨으로 표시
 a| 
.. 각 이벤트 세부 정보 페이지에서 이벤트의 원인을 확인하세요.
.. 이벤트를 선택하세요.
.. *확인*을 클릭하세요.
.. 적절한 시정 조치를 취하세요.
.. *해결됨으로 표시*를 클릭하세요.


|===
+
이벤트가 해결됨으로 표시되면 해당 이벤트는 해결된 이벤트 목록으로 이동됩니다.

. *선택 사항*: *메모 및 업데이트* 영역에서 이벤트를 어떻게 처리했는지에 대한 메모를 추가한 다음 *게시*를 클릭합니다.
